工作了几年之后，我发现自己一直不喜欢国内科技公司的加班文化以及工程师文化，所以前段时间一直在找国外的工作机会，经过几个月的面试，我最终决定到柏林的一家硬件公司工作，一方面因为他们像我一样热衷开源（产品的代码也是开源的），另一方面我喜欢他们的想法以及氛围。这篇文章我想分享这几个月的求职经历，希望能给一些想要到国外工作的读者提供一些建议。

心态

我把心态放在第一位，调整好心态，做好长期的规划和调查，才能过上愉快的程序员生活。国外工作看起来虽然美好，但是要克服的障碍也很多，例如文化差异以及饮食习惯，我第一次去欧洲的时候，发现晚上的地铁连站务员也没有，而且逃票的当地人到处都是。而且饮食嘛，同行的朋友都吃不惯，这点见仁见智。欧美晚上除了酒吧基本路上都没什么人，不过酒吧的歌倒是比国内好多了。如果之前没去过当地的话，我建议出国工作前最好先逛逛当地的留学生论坛或者华人论坛（美国的一亩三分地，德国的 abcdv）了解下华人眼中的情况。另外一点是对自己有信心，很多国家的程序员都非常紧缺，对学历和技术的要求其实没那么高，而且我觉得国人普遍比较勤奋，所以只要英文不错的程序员我觉得就能尝试下。

哪个国家 / 地区

每个国家都有自己的优点和缺点，亚洲虽然加班严重，工资较低，但是相对热闹。欧洲加班少，工资较高，不过税率也高。美国工资高，不过相对不安全（参考国家安全指数）。不同国家 / 地区对国外人才的签证要求不一样。我这几个月面试了美国，加拿大，英国，欧洲，新加坡，香港这几个地方，综合人才需求，签证难度，语言要求三方面得出到国家 / 地区工作难度：

很明显，这不是一两个月能完成的，持之以恒才是最重要的，有的时候英语与面试/沟通技巧比技术还要重要，所以要按自己的情况取长补短。

简历

如何写简历可以参考我们之前的文章如何写一份更好的简历，不过英文简历与中文简历要求不同，并不是直接拿中文简历翻译就可以， 例如，投美国的公司通常不允许出现照片，年龄，种族等个人资料。以我自己的经验来说，我自己有一个 Startup，在招一名 Remote 全栈工程师，发布了在 Hackernews 和 AngelList 上。我收到不少求职者的简历，同国内收到的简历一样，越优秀的人越重视自己的简历。而且这时候简历要和全世界的求职者竞争，尤为重要。竞争有多激烈呢？从学校来说，伯克利，哥伦比亚，加州大学都有。经验来说，前谷歌的员工，十多年工作经验的员工都有。简历关不过，其他什么都免谈，对自己的简历没信心的话可以使用我们提供的 简历 review 平台 。

有些公司会要求提供 Cover letter，也就是一段个人简介以及与该公司有关的要素。为什么要投这家公司，这个岗位。这点很重要，起码代表求职者认真对待这次面试，而且有了解过这家公司。我的求职方向是后端工程师以及全栈工程师，所以准备了两份简历，以及几份 Cover Letter 根据不同的情况投递，特别在乎的公司我会重新写一份 Cover Letter。至于如何写，可以参考这里的例句。

投简历实在花费了我大量的时间，有太多太多的求职平台，很容易会害怕错过某些好公司。而且有些平台不是上传简历就好，还需要填写额外资料以及信息，极为耗时。为此我自己还写了一个合适开源贡献者的求职平台，我的工作也是在上面找到的。抱着多了解，多锻炼的想法，我海投了超过一百家公司，面试了超过 20次。这里我总结了一些常用的平台，可以说基本锁定这些平台就好了。

获得面试机会

如果有认识的朋友可以内推，那当然是最好的途径，起码简历关能保证可以过。另外一个方式是找工作签证，例如德国的 jsv 签证，这种签证允许符合条件的人才到当地几个月找工作。除非国内的超一线大厂，不然大部分的公司他们也没听过。相对来说，国外的公司会更加看重开源项目的贡献以及英文博客内容。不止一次面试的时候他们让我选出最自豪的博客 / GitHub 项目给他们审核。所以，如果真的有出国的打算，现在开始打理 GitHub 以及博客绝对是一个好的决定。
